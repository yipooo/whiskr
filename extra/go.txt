## Modern Go Features (1.26+)
Always use the most modern, idiomatic Go features available when appropriate:
- Use `new(expr)` for pointer initialization of exact values (e.g., `new(300)` instead of `x := 300; &x`) (Go 1.26+).
- Leverage self-referential generics if it simplifies interface constraints (Go 1.26+).
- Use `sync.WaitGroup`'s `wg.Go(func() { ... })` instead of manual `wg.Add(1)` and `defer wg.Done()` (Go 1.25+).
- Use integer range loops (`for i := range n`) instead of classic `for` loops when applicable (Go 1.22+).
- Leverage `iter.Seq` and custom iterators (`for v := range seq`) for collections (Go 1.23+).
- Use `min()`, `max()`, and `clear()` built-ins. Use `any` instead of `interface{}`.

## Idiomatic Performance
**Memory & Allocations**
- Minimize heap allocations through sound data structures and API design, not clever hacks. Keep the new Green Tea GC (default in 1.26+) in mind, but still aim to reduce allocation pressure.
- Pre-allocate slices and maps with known capacities (`make([]T, 0, cap)`).
- Prefer passing slices as arguments to be appended to, rather than forcing the function to allocate and return a new slice.
- Avoid string allocations: use `[]byte` for manipulation, `strings.Builder` for concatenation, and the `bytes` package.

**Compiler Symbiosis**
- Facilitate Bounds Check Elimination (BCE) by hinting slice lengths to the compiler early (e.g., `_ = b[7]` before accessing `b[0]` to `b[7]`), but keep it readable.
- Write code that is friendly to escape analysis. Return values instead of pointers when structs are small.
